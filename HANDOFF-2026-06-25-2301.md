## Key Decisions Made This Session

1. **Tag filtering architecture:** Switched from `?tag=` + client-side JS to pre-rendered `/blog/tag/[tag]` static routes. Reason: the spec explicitly prohibited client-side SPA filtering and required "real `<a>` links read at build/request time." SSG static routes are the correct implementation.

2. **Credits page parsing:** Used regex-based build-time parsing in Astro frontmatter (`readFileSync` + targeted regex per entry type) rather than a config-driven or hard-coded approach. Fonts, GitHub Actions, and versioned npm packages each have their own parser because their NOTICES.md format differs.

3. **About page sticky card:** Applied `position:sticky; top:96px` to a wrapper `<div class="card-sticky">` from `about.astro`'s scope, not from inside `BusinessCard.astro`. This keeps the component clean and makes the sticky behaviour a layout concern of the page.

4. **About page narrative structure:** Story prose ("Why this blog exists", "What I'm building now") was moved inside the intro section's left grid column so it scrolls alongside the sticky card — matching the prototype exactly.
