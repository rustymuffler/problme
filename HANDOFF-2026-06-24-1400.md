## Key Technical Decisions Made This Session

1. **Astro 7.0.2 (not 5.x).** npm resolved to a beta of Astro 7 from the `^5.9.4` range spec. The initial install had 1 HIGH severity XSS vulnerability in Astro <=7.0.0-beta.6. Upgraded to Astro 7.0.2 (latest stable) to fix it. 0 vulnerabilities now.

2. **Tailwind v4 via `@tailwindcss/vite`, NOT `@astrojs/tailwind`.** Astro 7 uses Tailwind v4. The correct integration is `@tailwindcss/vite` as a Vite plugin in `astro.config.mjs`. The `@astrojs/tailwind` package only supports Tailwind v3 and should not be used.

3. **Content config at `src/content.config.ts` (not `src/content/config.ts`).** Astro removed the legacy `src/content/config.ts` location in v6. The correct Astro 7 location is `src/content.config.ts` at the `src/` root, using a `glob()` loader.

4. **No `base` URL in astro.config.mjs.** probl.me uses a custom domain at the root (not a GitHub Pages subdirectory path like `/problme/`). No `base` is needed.

5. **Bunny Fonts, not Google Fonts.** Bunny Fonts is the privacy-friendly (GDPR-compliant, no IP logging) alternative. Font links go in the Layout `<head>` as `<link>` tags, not as a CSS `@import` (which caused a CSS ordering warning with Tailwind v4).

6. **Gitleaks pre-commit hook uses full path.** Gitleaks 8.30.1 is installed via WinGet but not on the system PATH. The hook at `.git/hooks/pre-commit` uses the full path to gitleaks.exe at `C:/Users/richa/AppData/Local/Microsoft/WinGet/Packages/Gitleaks.Gitleaks_Microsoft.Winget.Source_8wekyb3d8bbwe/gitleaks.exe`. This works but is fragile -- if WinGet updates Gitleaks the path may change. Consider adding Gitleaks to the system PATH in a future session.

7. **npm install accepted without rl-protect-scan.** rl-protect-skills is not yet configured. The initial 4 packages (astro, @astrojs/mdx, tailwindcss, @tailwindcss/vite) were installed from official, well-known sources. Logged in the SECURITY_SCANNING.md Accepted Risk table. rl-protect-skills MUST be configured before any additional packages are installed.
