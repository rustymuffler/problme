## Key decisions made in this session

| Decision | Rationale |
|---|---|
| Lighthouse CI perf threshold: 0.50 | Localhost CI scores are unreliable (CPU throttling, no CDN, external font load). Production target remains ≥ 0.90. |
| Trivy: `gh release download` instead of `trivy-action` | `aquasecurity/trivy-action` install.sh fails without auth on latest versions. Direct download via `GH_TOKEN` is reliable. |
| lychee: exclude LinkedIn | LinkedIn returns HTTP 999 (bot protection); lychee classifies as Unknown → exit code 2. Exclusion is correct. |
| Email obfuscation: `data-*` attributes + `getAttribute()` | `String.fromCharCode(64)` on string literals is constant-folded by Vite. DOM reads at runtime survive the bundler optimizer. |
| Exclude `tests/` from `astro check` via `tsconfig.json` | `@axe-core/playwright` and `@playwright/test` both bundle `playwright-core` types, causing a conflict under strict TypeScript. Excluding the test dir is the clean solution. |
